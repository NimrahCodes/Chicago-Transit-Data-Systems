# Chicago-Transit-Data-Systems
High-performance SQL optimisation and data validation logic for 24M+ rows of transit telemetry.

Chicago Transit Operational Systems
Project Overview
This technical case study focuses on the challenges of managing and querying large-scale relational databases (24M+ rows).

Technical Focus
Query Performance: Implementing efficient aggregation logic to handle high-volume telemetry without system latency.

Data Validation: Identifying "orphaned" assets and null sensor pings within the transit network.

Relational Logic: Leveraging complex joins and indexing to extract actionable insights from raw operational logs.

Tools Used:
SQL (PostgreSQL)
Relational Database Management Systems (RDBMS)
